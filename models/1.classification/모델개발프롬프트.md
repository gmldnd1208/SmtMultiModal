방향성
1번모델 : csv 파일만을 인풋 받아서 이상감지 하는 모델 / threshold 계산 (1.anomaly_detection)
2번모델 : json, image 파일을 인풋 받아서 불량인지 아닌지 이진분류 (2.classsification)
3,4번 모델 : 각각 pr, sd 공정별로 불량의 종류 클래스를 구분하는 모델 (3.detection_pr, 4.detection_sd)
5번 모델 : 2,3,4번 모델에서 나온 불량 종류 결과와 센서 데이터를 결합해서 원인 도출


classification 개발 순서 (6-Step)
Step 1: 통제실 구축 (config.py)
목표: 모델 개발에 필요한 모든 파라미터들을 한 곳에 모아둡니다.

Step 2: 데이터 정제 및 목차 생성 (preprocess.py)
목표: 원본 데이터를 AI가 먹기 좋게 가공합니다.

할 일: 복잡한 JSON 파일들을 스캔하여 1~16번은 정상(0), 17~32번은 불량(1)으로 이진화합니다. 그리고 5초치 센서 데이터를 가벼운 .npy 파일로 변환한 뒤, 이미지 경로와 함께 metadata.csv라는 학습용 목차(CSV)를 만듭니다.


Step 3: 초고속 최적화 학습 엔진 (train.py)
할 일: 1. swin_transformer_v2.py를 불러와 눈(Vision)을 만듭니다.
2. timesnet.py를 불러와 귀(Sensor)를 만듭니다
할 일: Gradient Clipping , OneCycleLR하다가 성능 개선이 없으면 Early Stopping 학습 루프


Step 6: 성적 채점기 (evaluate.py)
목표: Test 데이터로 실력을 검증합니다.
할 일: 학습된 best_model.pth를 장착하고 F1-Score, Precision, Recall을 계산합니다. 특히 공장에서 가장 중요시하는 혼동 행렬(Confusion Matrix)을 출력하여 과탐/미탐을 분석합니다.
